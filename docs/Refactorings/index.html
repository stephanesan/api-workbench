Code Manipulation and Refactoring

Refactorings

Refactoring actions transform existing code leaving its functionality doing the same.

Refactorings are generally available in context menu in the “Refactoring” section and Palette tab of Details pane in the “Refactoring” category.

All refactorings are context-depended, that means they are only displayed in context menu or Palette when it suitable to apply a refactoring with regard to the current cursor position.

Following refactorings are available:

Extract resource type

This action allows selecting resource properties like methods and then creates a new resource type, puts those properties there and makes current resource to reference the newly created resource type.

Action context: this action is available in resource context.

Extracted element name field sets up new resource type name.

The left pane in the bottom contains the element to be moved to the right pane. Moved elements will be extracted. The buttons in the middle move the selected element or all elements.

In the following example get and post methods of /pets resource are moved to the newly generated Collection resource type.

Before:

#%RAML 1.0 
title: Pet shop
version: 1
baseUri: /shop
 
types:
  Pet:
    properties:
      name: string
      kind: string
      price: number
    example:
      name: "Snoopy"
      kind: "Mammal"
      price: 100
 
/pets:
  get:
    responses:
      200:
        body:
          application/json:
            type: Pet[]
  post:
    body:
      application/json:
        type: Pet
  /{id}:
    put:
      body:
        application/json:
          type: Pet
    delete:
      responses:
        204:

After:

#%RAML 1.0 
title: Pet shop
version: 1
baseUri: /shop
types:
  Pet:
    properties:
      name: string
      kind: string
      price: number
    example:
      name: "Snoopy"
      kind: "Mammal"
      price: 100
 
resourceTypes:
  Collection:
    get:
      responses:
        200:
          body:
            application/json:
              type: Pet[]
    post:
      body:
        application/json:
          type: Pet
 
/pets:
  type: Collection
  /{id}:
    put:
      body:
        application/json:
          type: Pet

        204:

Extract trait

This action allows selecting method properties and then creates a new trait, puts those properties there and makes current method to reference the newly created trait.

Action context: this action is available in method context.

Extracted element name field sets up new trait name.

The left pane in the bottom contains the element to be moved to the right pane. Moved elements will be extracted. The buttons in the middle move the selected element or all elements.

In the following example priceLessThen and priceMoreThen method query parameters of get method are moved to the newly generated FilterableByPrice trait.

Before:

#%RAML 1.0 
title: Pet shop
version: 1
baseUri: /shop
 
types:
  Pet:
    properties:
      name: string
      kind: string
      price: number
    example:
      name: "Snoopy"
      kind: "Mammal"
      price: 100
 
resourceTypes:
 
  Collection:
    get:
      responses:
        200:
          body:
            application/json:
              type: <<item>>[]
    post:
      body:
        application/json:
          type: <<item>>
 
  Member:
    put:
      body:
        application/json:
          type: <<item>>
    delete:
      responses:
        204:
 
/pets:
  type: { Collection: {item: Pet} }
  get:
    queryParameters:
      priceLessThen:
        type: number
      priceMoreThen:
        type: number
      petKind:
        enum: [bird, mammal]
 
  /{id}:
    type: { Member: {item : Pet} }

After:

#%RAML 1.0

version: 1

Move resource

This action allows moving resource into another resource in the same file.

Action context: this action is available in the resource context.

The displayed tree allows to select a resource to move current resource to.

Move content to other file

This action moves any RAML node to another file.

Action context: any RAML node.

Destination path - sets up destination file path relatively to the current file.

Beware: this action allows moving any node, the result may not always be valid.

Convert JSON schema to type

This action converts JSON schema to a type.

Action context: JSON schema directly located under schema attribute of a schema consumer. This action is not applicable to a top-level schema definition.

The is no dialog for the action, just select schema text or schema key and activate the action.

Expand type to JSON schema

This action converts type to JSON schema.

Action context: inplace (anonymous) type declaration or top-level type declaration.

The is no dialog for the action, just select type name or type key and activate the action.

Pull up

This action takes one or more type properties and pulls them up to a selected type’s supertype.

Action context: this action is available when a type declaration is selected. The type must have at least one supertype.

Select supertype combo allows choosing a target supertype.

Elements to move should be moved from the left panel to the right.

In this example color property is being pulled from Mammal type up to its Pet supertype.

Before:

#%RAML 1.0 
title: Pet shop
version: 1
baseUri: /shop
types:
  Pet:
    properties:
      name: string
      kind: string
      price: number
    example:
      name: "Snoopy"
      kind: "Mammal"
      price: 100
 
  Mammal:
    type: Pet
    properties:
      color:
        enum: [White, Black, Colored]
 
  Bird:
    type: Pet
    properties:
      wingLength: number

After:

#%RAML 1.0 
title: Pet shop
version: 1
baseUri: /shop
types:
  Pet:
    properties:
      name: string
      kind: string
      price: number
      color:
        enum:
          - White
          - Black
          - Colored
    example:
      name: "Snoopy"
      kind: "Mammal"
      price: 100
 
  Mammal:
    type: Pet
  Bird:
    type: Pet
    properties:
      wingLength: number

Extract supertype

This action takes one or more type properties, creates a new supertype for the current type, and puts the selected properties into the supertype.

Action context: this action is available when a type declaration is selected. The type must have no supertype.

Supertype name sets up a target supertype name.

Elements to move should be moved from the left panel to the right.

Extract library

This action allows reusing element from the current file in another ones by extracting them to the library.

Action context: API

Library path sets up the new library file name and path relatively to the current file. Library namespace sets up the namespace of this library to be imported as and used in the current file.

Elements to move should be moved from the left panel to the right.

Code manipulation actions

Comment node

This action comments the selected node and all its children.

Action context: any node.

Comment lines

This action comments the selected lines.

Action context: selection.

This action overrides standard Atom action.

Delete current node

This action deletes the selected node and all its children.

Action context: any node.

Create global schema

This action takes the locally defined schema, extracts it to a separate file, includes this file to a global (top-level) schema, and makes current element to reference the newly extracted schema.

Action context: inplace schema definition.

Schema name - sets up both new schema file name, and the name of the top-level schema, which will reference this file.

Extract overlay

This action extracts selected elements declarations to a new overlay.

Action context: API

Overlay path sets up the new overlay file name and path relatively to the current file.

Elements to move should be moved from the left panel to the right.

Modify overlay

This action extracts selected elements declarations to an existing overlay from its master.

Action context: Overlay file having a valid master reference.

Elements to move should be moved from the left panel to the right.

This action filters out the elements of the master, which already exist in the current overlay, only the new elements from the master, which can be added to the overlay are displayed.
